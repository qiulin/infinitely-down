## 一、整体分期思路

- **近期（0–3 个月）**：做最小可行版本（MVP），验证核心玩法 + 基础广告变现，尽快上线拿数据。  
- **中期（3–9 个月）**：在验证通过的前提下，铺开社交、养成和赛季，形成 **留存 + 付费 + 社交裂变** 的闭环。  
- **远期（9–18 个月）**：做品牌和生态层面建设，如 UGC、直播联动、IP 合作，冲击头部产品。

---

## 二、近期开发计划（MVP：0–3 个月）

**阶段目标：**  
上线可玩的核心版本，验证“下落玩法 + 广告复活/奖励”的基础数据（留存、时长、eCPM）。

### 1. 功能范围（建议全部为 P0 / 必做）

1. **基础框架**
   - 微信登录、用户唯一 ID 绑定
   - 本地+简单云端存档（最高层数、金币、基础设置）
   - 基础 UI：主界面、开始游戏、结算界面、设置界面

2. **核心玩法**
   - 竖版下落核心逻辑：重力、碰撞、掉出屏幕判定
   - 操作：左右移动 + 基础跳跃（轻点/长按可暂缓，先简化为单一方案也可以）
   - 平台类型（首发即可）：
     - 普通板
     - 移动板
     - 破碎板
   - 场景与难度：
     - 无尽模式 1 套场景
     - 按“10 层为一小段”做简单难度渐进（平台间距、移动速度递增）
   - 基础结算：本局下到的层数、获得金币

3. **基础经济与角色**
   - 软通货：金币  
     - 来源：通关结算、首发任务  
     - 用途：解锁/升级 1–2 个简单属性（如移动速度略微提升）
   - 角色：  
     - 2 个免费基础角色（数值一致，仅外观略有差异或完全一致）

4. **基础社交（微信最小接入）**
   - 好友排行榜（显示最高层数）
   - 炫耀分享：达到新纪录时，生成分享卡片（分数+头像+文案），点击回到游戏

5. **基础商业化（IAA 为主）**
   - 激励视频广告：
     - 死亡后观看广告复活 1 次
     - 结算双倍金币
   - 插屏广告：
     - 每若干局出现 1 次（如 5 局一次），可后端调参
   - Banner 广告：
     - 主界面底部常驻（不影响操作区域）

6. **数据与运营基础**
   - 埋点：
     - 新增用户、回访用户、DAU、局数、平均时长
     - 平均死亡楼层、死亡原因（掉空、被顶、踩破碎板等）
     - 激励视频播放次数、完播率
   - 新手引导：
     - 简单 3–5 步：移动、跳跃、躲陷阱
   - 一个经典模式：
     - “100 层挑战”作为新手/老玩家情怀入口（可先做成简单版，时间不限）

### 2. 关键里程碑（可按 2 周一小步）

- 第 2 周：玩法原型跑通（内部可以完整玩一局）  
- 第 4 周：核心玩法 + 基础 UI 完成，开始内部测试  
- 第 6 周：广告、排行榜接入完毕，小规模灰度  
- 第 8–10 周：提交审核 + 正式上线  
- 第 12 周：根据首轮数据，做一次 **难度 + 广告节奏** 调优版本

## 三、中期开发计划（增长与生态：3–9 个月）

**阶段目标：**  
在核心玩法稳定的前提下，加入 **多模式 + 角色养成 + 社交玩法 + 赛季 + IAP**，建立长期留存和营收能力。

### 1. 核心方向

1. **丰富玩法，增加可玩深度**
   - 新增平台与陷阱：
     - 弹簧板、传送带平台、冰面平台、钉刺平台、隐藏平台
   - 动态威胁：
     - 上升尖刺墙 / 熔岩
     - 简单横向飞来障碍
   - 无尽模式难度曲线精细化（根据近期数据调参）

2. **多人模式 & 微信社交深度利用**
   - 好友接力赛：
     - 完成一段后生成“接力层”，分享给好友，好友继续往下
   - 群排行榜：
     - 以微信群为单位显示 TOP 成绩，支持“一键挑战 TA”
   - 实时擂台赛（轻量 PVP）：
     - 2–4 人匹配，同一张图同步下落，看谁活得更久/下得更深  
     - 首期可做简单同步（玩家影子、非传统高实时对战）

3. **角色成长与养成**
   - 角色等级：
     - 使用金币/经验升级，提升少量基础属性（移动速度、最大生命值等）
   - 角色星级/被动技能（可分步上线）：
     - 如：金币加成、暴击层数加成、暴击得分等
   - 装饰向皮肤：
     - 不影响对战公平的前提下，可附带轻微 PVE 加成

4. **完善经济 + 上线 IAP**
   - 软通货：金币用途扩展（角色升级、基础抽奖）
   - 硬通货：钻石上线
     - 用于购买：高级皮肤、稀有角色、复活包、月卡等
   - 商品设计（示例）：
     - \￥1–3：新手礼包（钻石+复活币+皮肤）
     - \￥18–25：月卡（每日钻石 + 体力/复活奖励）
     - \￥30：赛季通行证（见下）
   - 赛季通行证系统：
     - 每 30–45 天一季
     - 免费轨道：金币、复活币、基础道具
     - 付费轨道：限定皮肤、头像框、称号、钻石

5. **运营与活动体系**
   - 每日/每周任务：
     - 玩 X 局、达到 Y 层、观看 Z 次广告等
   - 每日试炼：
     - 全服统一地图参数，限次数，作为回流抓手
   - 节日活动：
     - 春节 / 中秋 等主题皮肤、特效层

### 2. 中期关键版本节点（示例）

- 约第 14 周：  
  - 上线接力赛 + 群排行榜 + 每日任务
- 约第 18 周：  
  - 上线公会/战队雏形（3–20 人）、公会总里程榜
- 约第 20–24 周：  
  - 上线赛季通行证 + 正式 IAP 商品  
  - 第 1 个完整主题赛季（如“地心试炼”）
- 约第 26–36 周：  
  - 实时擂台赛稳定运营  
  - 至少 2–3 个节日活动跑通  
  - 广告与付费数据趋于稳定，开始 A/B 测试（价格、广告频次、道具组合）

## 四、远期开发计划（品牌与生态：9–18 个月）

**阶段目标：**  
在产品趋于稳定后，通过更高级的内容体系和生态联动，将游戏从“赚钱产品”升级为“IP 与平台级资产”。

### 1. 深度内容与 UGC

1. **地图编辑器 / UGC**
   - 玩家可自定义平台组合、陷阱分布，生成“自研关卡”
   - 内部审核通过后，对外开放挑战
   - 排行榜：热门地图 / 高评分地图 / 高通过率地图

2. **进阶任务与剧情线**
   - 长线任务线（例如“从地表到地心”的故事）  
   - 章节解锁：通过指定层数或任务解锁特殊关卡 / 皮肤

3. **特殊事件层 & 主题世界**
   - 特定时段开放特殊地形（如“低重力月球层”“反向重力层”）
   - 与现实节日、品牌活动联动

### 2. 直播与视频号联动、社交生态深化

1. **视频号直播联动**
   - 主播玩游戏时，观众可：
     - 送礼触发陷阱/增强难度
     - 赠送道具帮助主播过关
   - 通过 CPS 或视频号激励政策获取额外收益

2. **更强的公会 / 社区系统**
   - 公会战、跨公会联赛、公会赛季排名
   - 公会专属任务、徽章、皮肤

3. **玩家社区建设**
   - 内置简单“关卡评论”、点赞系统
   - 官方维护的攻略、优质关卡推荐列表

### 3. IP 与多维变现

1. **IP 联动**
   - 与动漫/影视/品牌联动推出：
     - 联动角色
     - 联动皮肤
     - 联动活动层（如某品牌主题关卡）
2. **周边与外延产品**
   - 表情包、头像挂件、周边实物（可通过公众号/小商店售卖）
3. **跨平台扩展**
   - 微信 PC 端适配加强
   - 评估是否需要独立 APP 或其他平台轻度移植版（保持玩法一致、运营节奏可衔接）

### 4. 远期版本关键节点（示例）

- 约第 40 周：UGC 编辑器内测 + 小范围开放  
- 约第 44–52 周：首个大型 IP 联动活动  
- 约第 60 周：直播互动玩法成熟（视频号联动常态化）  
- 约第 72–84 周：形成“赛季 + 公会战 + 直播 + IP 联动”固定节奏

## 五、如何实际落地这套分期计划

如果你要拉团队去做，建议：

1. **先锁近期范围，做清单化拆解**  
   把“近期阶段”所有 P0 功能拆成任务卡，估算人日，排入 2 周迭代节奏。

2. **中期、远期只做“方向锁定 + 粗粒度排期”**  
   不急着细化每一个功能，等 MVP 数据回来后再对中期功能做删减或调整优先级。

3. **每结束 1 个主要迭代就做一次复盘**  
   对照本阶段 KPI（留存、时长、eCPM、付费率、分享率），决定：
   - 哪些中期功能要提前
   - 哪些可以取消或大幅简化
