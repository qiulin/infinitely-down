# 无限向下

> 基于经典游戏“是男人就下100层”的核心玩法，结合当前微信小游戏生态、社交能力与混合变现模式，设计的一款可长线运营的竖版下落类休闲竞技小游戏完整方案。

---

## 一、产品整体定位

### 1.1 产品概述

- **名称（暂定）**：无限向下
- **类型**：竖版下落类休闲 + 轻度竞技 + 社交对战  
- **节奏**：单局 60–120 秒，适配碎片化时间  
- **平台**：微信小游戏（主打手机端，同时支持微信PC端）  
- **目标用户**：
  - 有童年“下100层”记忆的 20–35 岁男性为核心圈层
  - 喜欢轻度竞技、排行榜攀比的泛休闲用户（18–45 岁）

### 1.2 核心卖点

1. **经典升级**：保留“是男人就下100层”紧张刺激的下落体验与高难度“反人类”感，但做成适应触控与短局时长的节奏化设计。  
2. **强社交属性**：深度接入微信“好友排行榜”“社交擂台赛组件”等能力，通过实时对战、群排行、接力挑战等方式放大裂变。  
3. **混合变现**：广告（IAA）+ 内购（IAP）+ 视频号直播联动，实现多元收入。  
4. **长线运营**：赛季制 + 角色养成 + 皮肤收集 + 公会赛，兼顾短期爆发与长期留存。

## 二、玩法设计

### 2.1 核心玩法（承接经典）

**基础规则（继承“是男人就下100层”的精髓）**：

- 玩家控制角色左右移动，在不断向下移动的场景中踩各种平台往下跳。
- 若角色被向上推顶出屏幕、掉入深渊或踩到陷阱（如钉刺等）导致血量清零，则本局结束。[1][2][8][14]
- 目标：在限定时间 / 条件下下到尽可能多的楼层，或挑战特定目标层（如 100、200、500 层）。

**操作方式（移动端优化）**：

- 左右滑动：控制角色水平移动。
- 轻点屏幕：小跳（短位移，下落更可控）。
- 长按屏幕：蓄力高跳（适合跨越大间隙或危险平台）。
- 可选：开启重力感应模式（左右晃动手机控制移动），用于活动或趣味玩法。

### 2.2 场景与关卡设计

#### 2.2.1 平台与陷阱类型

在经典基础上扩展，保证节奏多样和策略性：

- **基础平台**：  
  - 普通板：稳定不动，无特殊效果。  
  - 移动板：左右移动，需要预判移动轨迹。  
  - 破碎板：踩上后 0.5 秒破裂，需要迅速弹离。  
  - 弹簧板：提供高跳，适合快速跨越多层。  
- **进阶平台**：  
  - 传送带平台：带有方向箭头，会持续推着角色移动。  
  - 冰面平台：摩擦力小，难以停下。  
  - 钉刺平台：踩上扣血，甚至秒杀。  
  - 隐藏平台：接近时才显形，可放在高分区增加可玩性。  

- **动态威胁**：
  - 上升尖刺墙 / 熔岩：强制玩家持续向下，形成压迫感。
  - 横向飞来的障碍：需配合小跳躲避。
  - 追击怪物：逼迫玩家加快节奏，增加紧张感。

#### 2.2.2 难度曲线

- 以“10 层为一小段，50 层为一大段”的节奏，逐步提高：
  - 平台面积变小、间隔变大。
  - 移动速度、陷阱密度增强。
  - 加入更复杂的动态陷阱组合。  
- 通过后台数据（平均达成层数、死亡原因统计）动态微调难度，控制大部分玩家的平均层数在“有一点挑战但经常能进步”的区间，如 80–150 层。

### 2.3 模式设计

#### 2.3.1 单人模式

1. **无尽模式（核心模式）**  
   - 无固定终点，下得越深得分越高。  
   - 适合竞赛、排行榜、长线挑战。  

2. **100 层挑战（经典模式）**  
   - 目标：在有限时间内（如 60 秒 / 90 秒）尽可能快地下到第 100 层。  
   - 适合作为新手引导与“经典致敬”内容，完成挑战可解锁称号“真男人”。  

3. **每日试炼**  
   - 每日固定一套关卡参数（所有玩家相同），增加公平性与策略研究空间。  
   - 仅 3 次机会或需消耗体力，拉高回访率。  

#### 2.3.2 社交 / 对战模式（微信特色）

1. **好友接力赛**  
   - 我先玩一局停在某一层，系统生成“接力点”。  
   - 我分享到好友 / 群，好友点击可从该层继续往下挑战，最终成绩为两人总层数。  
   - 有助于自然分享裂变，增加“帮人一把”的情感动机。  

2. **实时擂台赛（对战组件）**  
   - 利用微信提供的“社交擂台赛”能力，2–4 人同步匹配。  
   - 所有人同版图实时下落，可以看到其他玩家的影子或小头像。  
   - 规则：谁先死亡 / 被顶出视野则淘汰，剩余玩家排名；或在限定时间内看谁下得更深。  
   - 用于赛季排位、周末比赛、视频号直播对战。  

3. **群排行榜与 PK**  
   - 以微信群为维度，展示群内 Top10 成绩。  
   - 支持“一键发起群挑战”：成员可依次尝试超越发起者的成绩。

## 三、数值与成长设计

### 3.1 角色系统

#### 3.1.1 角色类型

- **普通角色**：初始免费 1–2 个（男 / 女 / Q 版英雄），数值平衡但无特殊技能。
- **稀有角色**：通过抽卡、活动或充值解锁，拥有被动技能，如：
  - “起跳更高”“落地减伤”“更容易触发暴击得分”等。
- **限定联动角色**：与品牌、IP 联动推出，主要卖点在外观和特效。

#### 3.1.2 成长维度

- **等级**：消耗金币 / 经验升级，提高基础属性（移动速度、最大生命值、道具效果时间等）。  
- **星级**：通过重复获取同角色进行升星，解锁额外被动效果或特殊动作效果。  
- **皮肤**：纯外观或带轻微加成（不影响公平对战模式，可以在 PVP 中屏蔽属性）。  

### 3.2 道具与技能

#### 3.2.1 战斗道具（局内）

局内随机掉落或开局携带，使用后仅本局生效：

- 护盾：抵挡一次伤害。  
- 磁铁：一段时间自动吸取金币 / 道具。  
- 时间减缓：短时间内场景整体减速，便于操作。  
- 楼层跳跃：直接瞬移下落 5–10 层。  
- 复活币：死亡后原地复活一次（也可通过激励视频获得）。

#### 3.2.2 被动技能（局外）

与角色绑定或通过天赋树解锁：

- 更高的金币收益。  
- 提高稀有平台 / 宝箱触发概率。  
- 增加强化道具持续时间。  

### 3.3 经济系统

- **软通货（金币）**：  
  - 来源：通关奖励、任务奖励、广告激励。  
  - 用途：角色升级、解锁基础皮肤、日常抽奖。  

- **硬通货（钻石）**：  
  - 来源：充值、活动、赛季奖励。  
  - 用途：高级抽卡、限定皮肤、赛季通行证升级、购买高价值复活包等。

## 四、社交与裂变设计

### 4.1 微信生态能力利用

- **好友关系链**：用于构建好友排行榜、接力挑战、邀请奖励。  
- **微信群**：群内榜单、群挑战赛、群内红包活动。  
- **视频号**：开通“看直播玩游戏”能力，在视频号直播中嵌入小游戏，实现边玩边导流边变现。  
- **公众号**：更新活动公告、攻略，承接自然搜索流量。

### 4.2 核心社交机制

1. **好友 / 群排行榜**  
   - 日榜 / 周榜 / 总榜，多维度展示。  
   - 榜单带有“一键挑战 TA”按钮，点击即进入同模式同条件局。  

2. **邀请裂变任务**  
   - 邀请 1/3/5/10 个好友体验游戏，逐级解锁：  
     - 复活币、限定道具、头像框、限定皮肤碎片。  
   - 防刷机制：要求新玩家实际完成 1 局游戏或达到 X 层。  

3. **公会（战队）系统：中后期开放**  
   - 3–30 人组成战队，可参与战队赛、战队总里程挑战。  
   - 公会贡献用于解锁战队专属徽章、装饰、称号。  

4. **“晒成绩”分享卡片**  
   - 玩家达到个人新高层数，自动弹出分享引导，可一键生成战报海报：  
     - 显示头像、昵称、最高层数、击败多少 % 的全国玩家等字样，提高炫耀动力。

## 五、商业化设计（变现方案）

结合当前小游戏主流模式与微信对 IAP、广告的支持，采取 **混合变现（广告 + 内购）** 策略。

### 5.1 广告变现（IAA）

#### 5.1.1 激励视频广告

- 触发场景：
  - 死亡后“观看广告免费复活一次”。  
  - 结算界面“看广告得双倍金币 / 额外道具”。  
  - 商城每日免费抽奖机会通过广告刷新。  
- 控制频率：单局最多 2–3 次，每位用户日均不超过 10 次，以防疲劳。  

#### 5.1.2 插屏 / 原生广告

- 出现位置：
  - 模式切换 / 多局连续失败之后。  
- 频率：每玩 5 局左右出现一次，不强制点击。  

#### 5.1.3 Banner 广告

- 位于主界面底部 / 次要页面，不影响操作区域，提供低干扰的持续曝光。

### 5.2 内购变现（IAP）

#### 5.2.1 核心付费点

- **皮肤与外观**：限定皮肤、特效轨迹、角色语音包。  
- **成长加速**：如金币加成卡、经验加成卡（控制不影响公平 PVP）。  
- **道具包**：复活币大礼包、高级局内道具礼包。  
- **赛季通行证**：
  - 免费 + 付费两条奖励轨道。  
  - 付费通行证可解锁：专属皮肤、头像框、动态称号、更多钻石。  

#### 5.2.2 商品示例与定价（示例，后续需 A/B 调优）

| 商品类型         | 价格（人民币） | 说明                          |
|------------------|----------------|-------------------------------|
| 新手礼包         | \￥1.0–3.0      | 钻石 + 复活币 + 基础皮肤     |
| 月卡             | \￥18–25       | 每日领取钻石 + 体力/复活奖励 |
| 高级抽卡（10连） | \￥30–68       | 抽角色 / 皮肤 / 稀有道具     |
| 赛季通行证       | \￥30          | 赛季周期约 30–45 天          |

### 5.3 视频号直播与赞助

- 支持视频号主播在直播中玩游戏，观众可通过送礼物、赞助的方式触发：
  - 增加主播游戏难度（更多陷阱、加速）。  
  - 或给主播发放强力道具，形成互动效果。  
- 开发与发行方可与主播进行分成合作，共享直播带来的增量收入与曝光。

## 六、运营与活动设计

### 6.1 上线期（0–3 个月）

1. **冷启动期（0–1 月）**  
   - 利用微信 IAP 新游首发激励政策争取流量位与现金激励。  
   - 重点优化新手引导、首日留存、3 日留存。  
   - 投放少量朋友圈 / 微信广告测试转化漏斗。

2. **爆发期（1–3 月）**  
   - 联合视频号 KOL 举办“真男人 100 层挑战赛”，设置现金奖金 / 实物奖品。  
   - 推出首个赛季主题（如“地心试炼”），伴随通行证上线。  
   - 开启“战队招募”活动，鼓励用户拉人建公会。

### 6.2 长线运营（3 个月后）

- **赛季更新**：每 1–2 个月一个主题赛季，引入新平台机制、新皮肤、新地图背景。  
- **节日活动**：春节“红包层”、中秋“月球重力层”、国庆“烟花特效层”等。  
- **跨界联动**：与零食、饮料、影视、动漫 IP 联动推出“联动角色 / 皮肤”，线上线下互导。  

### 6.3 数据指标与优化

重点关注：

- 核心：1 日 / 7 日 / 30 日留存，平均在线时长，人均日局数。  
- 商业：付费率、ARPPU、广告填充率与 eCPM。  
- 社交：分享率、被分享打开率、群挑战参与率。  

通过埋点分析死亡原因、关卡层数分布，对平台组合进行动态调参。

## 七、技术与实现要点（概略）

- **前端引擎**：Cocos Creator 或 Laya，输出微信小游戏包，控制首包体积 ≤ 4MB。  
- **后端服务**：基于云函数 / Serverless，实现排行榜、对战匹配、存档等功能。  
- **数据安全与反作弊**：  
  - 关键数值（分数、货币）在服务端校验。  
  - 检测异常操作（如极短时间内异常高层数）并封号 / 降权。

## 八、方案总结（对原问题的直接回应）

1. **玩法上**：  
   - 核心是对“是男人就下100层”的移动端重构：仍是躲避陷阱、踩平台下落的高压反应游戏，但通过触控优化、平台多样化和节奏分层让游戏更适合短时游玩与长期深钻。  

2. **微信小游戏特点的利用**：  
   - 使用好友排行榜、群排行榜、接力赛、实时擂台赛等社交组件，把原本单机的“虐心小游戏”升级成“朋友圈集体较劲”的轻竞技产品。  
   - 通过分享卡片、群挑战、视频号直播联动放大自然增长与社交裂变。  

3. **盈利方式**：  
   - 以激励视频 + 插屏为主的广告变现，嵌在复活、结算、免费抽奖等高转化节点。  
   - 以内购皮肤、角色、通行证为主的 IAP 体系，辅以成长加速与道具礼包，形成耐玩又不过度氪金的付费结构。  
   - 后期接入直播、联名周边等额外变现手段。  

4. **整体产品形态**：  
   - 一款 **“上手 10 秒就会玩、但想下到更深楼层要长期练习”** 的竞技向休闲小游戏。  
   - 既满足老玩家对“是男人就下100层”的情怀，又符合 2025–2026 年微信小游戏生态下，对社交性、可运营性与商业化的现实要求。
